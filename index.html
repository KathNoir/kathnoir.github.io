Feature · Syntax

Multiple Negation

Multiple negation — the use of more than one negative marker in a single clause — is a well-documented feature of African American Vernacular English. This tool automatically detects sentences containing multiple negators, scanning both forward and backward within each clause to identify co-occurring negative words like no, nothing, never, and n't.

Feature · Aspect

Habitual Be

Habitual be is a distinctive aspectual marker in African American English, expressing actions or states that occur regularly over time. This tool uses a rule-based system — examining the part-of-speech context around each instance of be — combined with an n-gram language model and a trained machine learning classifier to automatically identify habitual be in naturalistic speech data.

Now the "Multiple Negation" tagger can run. This tagger was not written by me, and is very simple. Step 4: Run Habitual Be Tagger

Now the "Habitual Be" tagger can run. This tagger works by locating "be" in a sentence, and uses n-gram similarity to see if the be is habitual or not. For a "be" to be habitual, it needs to be surrounded by certain words in the sentence like "I" or other words that the model detects as probable.

Background · Context

About the Project

African-American English is often called "improper English" or "ungrammatical". However, we have learned that AAE has features with specific grammar rules that won't work if used incorrectly. This project aims to document African-American stories, and the AAE features within them.

The Story

ALFAH, or "Annotating Linguistic Features in African American Oral Histories", is a project dedicated to preserving and studying the oral histories of African American communities. The data comes from the Samuel Proctor Oral History Program at the University of Florida, which houses a rich collection of first-person narratives told in the speakers' own voices.

My contribution focuses on the computer science side of this research. The code automatically identifies grammatical features characteristic of African American Vernacular English, which is a rule-governed dialect with its own systematic grammar, not a deviation from "standard" English. Two features are examined in particular: habitual be, which marks recurring actions ("she be working late"), and multiple negation, the use of more than one negative marker in a single clause ("I ain't never seen nothing like that"). By showcasing the features of the dialect and how its used grammatically in sentences, I hope to show that African American English is not "improper English" and will be documented appropriately for years to come.

Process · Tools

Making this Website

This website was built using Claude AI, on Sonnet 4.6.

The Tool

To built this website, I decided to use Claude AI. Out of all the LLM options in the world right now, I felt that Claude was the least terrible (morally) and the best in terms of coding. I let it use whatever framework it wanted, but since most websites are built using HTML and CSS, I had a feeling it would use those anyways.

Design Decisions

As for the design, I allowed Claude to give me whatever design it wanted as long as it followed my instructions. The prompt was as follows:

"Please make me a website to showcase some images and text. If you want to make it fancy, make it fancy. Please use a color palette using d8e2dc, ffe5d9, ffcad4, f4acb7, 9d8189. Id like for each div for the image and text to be on opposite sides, then flip for each div as it goes down the page."

The initial website looked good, but I continued asking for more features to make it look even better and easier for me to work on. Some examples are adding easy image named, and adding the NavBar at the top, which I prompted it to make with this:

"Please make it super easy for me to take pictures from a folder called "images" in the same website folder, and use easy names like "image_1.png" as placeholders. Please also make a navbar with four tabs. One tab is "Home", then "Multiple Negation", then "Habitual Be", then "About the Project"."

Then when it made it an ugly "hamburger" menu, I told it this:

"Can you make the navbar not be behind a hamburger menu. I want it to be visible at all times as you scroll down."

And continued for quite some time, with more small changes, until the website is what it is now.

What I Learned

When I started using Claude for more nuanced tasks, like making the images zoom in or out in a certain way, it became difficult for it to understand what I wanted. In the interest of time and tokens, I decided to leave some images a bit uglier than others since I couldn't get Claude to do the images how I wanted it to. Whole Claude AI si good at various web design aspects, it had a lot of trouble with the more nuanced parts, which may explain why vibe-coded websites are so easy to spot.
